# Dashboards (concept)

Dashboards in this system are designed for operational clarity and traceability across cases and sessions.

They answer:

1) What is the current state of a case?
2) What happened in the most recent sessions?
3) Which protocols are being used (and how often)?
4) How is global session severity (TEIA, 1–5) behaving over time?

## Client / case-level dashboard (typical widgets)

### A) Next session monitoring
- Next scheduled session date/time (when available)
- Days until next session (countdown)

### B) Case activity summary
- Total sessions recorded
- Current cycle and current cycle session (if cycle markers are used)

### C) Recent sessions table
A filtered table showing:
- date/time
- session type
- global session severity (1–5)
- N-level tier (N1/N2/N3/Mixed/Unclear)
- protocols used (linked)

### D) Global session severity trend (1–5)
A line chart (by week or by session order) showing:
- average or latest global session severity
- useful for monitoring stability vs volatility

### E) Protocol usage (optional)
- top protocols used in the last X sessions
- protocols most commonly used in higher-severity sessions (4–5)
