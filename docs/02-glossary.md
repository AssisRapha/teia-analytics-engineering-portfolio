# Glossary (domain terms)

## TEIA
A structured methodology to develop stability for spoken English under pressure (professional contexts).

## Case
A longitudinal unit of follow-up for one client (or one journey).
In data terms: one Case has many Sessions.

## Session
A single interaction / appointment / execution event.
In data terms: a Session is the atomic event record, with timestamp and observations.

## Protocol
A repeatable procedure (a “playbook”) applied during sessions to target a function such as entry, continuity, repair, conciseness, etc.
In data terms: Protocols are a catalog, and each Session can use multiple Protocols.

## N-level (N1 / N2 / N3)
Internal TEIA classification tiers used to describe the predominant instability pattern observed in a session and guide decision-making.
In this public repo, treat these as **tier labels** (not as full theoretical publication).

## Global session severity (TEIA, 1–5)
TEIA scale for the global severity of observable instabilities in a session, used for operational monitoring and capture governance.

- **1 (🟢) — Discrete**: mild, localized signal; does not meaningfully compromise global continuity.
- **2 (🟡) — Light**: instability is present but easily manageable with low intervention.
- **3 (🟠) — Moderate**: clinically relevant impact; reversible under clear constraints/support.
- **4 (🔴) — High**: significant continuity/form compromise; requires frequent intervention.
- **5 (⬛) — Severe**: high instability / near-collapse of sustained communicative stability.

Note: this repository uses the 1–5 TEIA scale as the standard.
