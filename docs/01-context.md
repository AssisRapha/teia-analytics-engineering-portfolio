# Context

This repository documents a data-ops / analytics engineering portfolio project based on a real operational system used to run a service domain (TEIA).

**TEIA** is a structured methodology focused on building stability for spoken English under high-pressure professional contexts (e.g., meetings, executive interactions, high exposure, time pressure).

The goal of this repo is **not** to publish client content or the full methodology. The goal is to demonstrate:

- data modeling for a real operational domain (cases, sessions, protocols)
- governance (standardized capture and terminology)
- dashboards / operational monitoring concepts
- a safe public demo dataset (fictional)

Key outputs:
- a clear data model (entities + relationships)
- a demo dataset (CSV) that reproduces the model without exposing real data
- documentation that makes the domain understandable to non-TEIA readers
